# Substack Drafts  - GitHub Edition

> **Bookmark this page:** [Latest Draft](https://github.com/mphinance/mphinance/blob/main/docs/substack/latest.md)
>
> This directory holds Substack drafts as markdown files hosted on GitHub Pages.
> Michael reviews, rewrites in his voice, then publishes manually on Substack.

## Structure

```
docs/substack/
├── README.md           # This file
├── latest.md           # Always the current draft (static bookmark URL)
├── latest.html         # Auto-generated HTML version (GH Pages)
├── musings/            # "Michael's Musings"  - weekly personal diary drafts
│   └── YYYY-MM-DD_topic.md
├── dossier/            # Daily dossier drafts (auto-generated by pipeline)
│   └── YYYY-MM-DD_dossier.md
└── archive/            # Published posts moved here
    └── YYYY-MM-DD_title.md
```

## Workflow

1. **Pipeline runs** → generates `dossier/YYYY-MM-DD_dossier.md`
2. **Agent/Michael writes** → creates `musings/YYYY-MM-DD_topic.md`
3. **Draft manager** copies the newest one to `latest.md`
4. **Michael reviews** at the static GitHub Pages URL, rewrites in his voice
5. **Michael publishes** on Substack manually
6. **RSS checker** detects the post on `mphinance.substack.com/feed/`
7. **Draft manager** moves `latest.md` to `archive/` and promotes the next draft

## RSS Dedup

The draft manager checks the Substack RSS feed for fuzzy title matches.
If a draft's title (or close variant) appears in the feed, it's considered "published"
and gets archived automatically on the next pipeline run.
