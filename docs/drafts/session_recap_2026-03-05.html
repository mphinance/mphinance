14 Hours, 7 API Endpoints, and a Job Application I Didn't Plan

The day I accidentally built an entire analytics platform, sent a cold email to a billion-dollar company, and still didn't trade.
Michael Hanko · March 5, 2026 · Momentum Phinance

I need to tell you about today because it's the kind of day that reminds me why I do this.

I sat down this morning with a list of small fixes. Broken TradingView links, a stale API, some CSS tweaks. Normal maintenance stuff. Fourteen hours later I'm staring at a fully rebuilt analytics API, a methodology showcase page I'm actually proud of, a landing page that finally looks like it belongs to someone who gives a shit, and a sent email in my inbox from recruiting@substackinc.com.

I didn't plan any of it. That's the point.

The API That Wouldn't Stay Dead

My Ghost Alpha API has been a zombie for weeks. Stale container, wrong port, old code. This morning I killed the process and rebuilt the whole thing from scratch. Seven endpoints, all live:

  • VoPR-graded CSP setups — every cash-secured put candidate with strike prices, premiums, Greeks, and a clear A/B/C/F grade
  • Daily momentum picks — 9-factor scored, top 3 medal winners, the same data that feeds my landing page
  • 3×3 daily setups — Day Trade breakouts, Swing pullbacks, CSP income plays, three picks per style
  • Market news aggregation — CNBC, MarketWatch, Yahoo Finance, Investing.com, all in one feed
  • Market regime detection — VIX context, hedge suggestions, bull/bear/neutral classification
  • Deep dives — full technical + fundamental data for any tracked ticker

Dockerized. Swagger docs. CORS open. Try it right now. It's free. For now.

The VoPR Showcase

I've been running VoPR (Volatility Options Pricing & Range) for months but never told anyone how it works. Today I fixed that.

Built a full methodology showcase page — the 4-model realized vol composite, how VRP ratios expose overpriced premium, the grading system. I showed enough to impress, not enough to replicate. The live data table is right at the top.

Here's the funny part: every single setup today got an F grade. Every one. Because we're in a HIGH vol regime with VRP ratios under 1.0. The system is literally saying "don't sell premium today, genius." And that's exactly why it works — it saves you from the trades that look good on a screener but will eat you alive.

Most options screeners sort by yield and call it analysis. That's like sorting basketball players by height and calling it scouting.

Secrets, Vaults, and 31 API Keys

Side quest: I consolidated every API key I own — scattered across 6 machines, buried in .env files, saved in random text documents — into one vault. VaultGuard. Firebase Firestore for cloud sync. FastAPI + MCP server for programmatic access. CLI for quick lookups.

Set up a new machine: python3 secrets_server.py --sync-down. Done. 31 keys. One command.

This is the kind of infrastructure nobody sees but makes everything else possible.

The Landing Page, Finally

mphinance.com used to show all 10 daily picks. Now it shows 3 — Gold, Silver, Bronze. Clean. Plus a new 3×3 setup grid: Day Trade, Swing, and CSP columns, each with 3 ranked picks and a 🏆 for the top pick. Category accent colors. Live data from the pipeline every morning.

I also added VaultGuard to the product grid because why not. If people ask about it, maybe there's a product there.

The Substack Email

This is the part I can't believe I actually did.

I was looking at my Substack dashboard and noticed ASCII art in the browser console: "SUBSTACK WANTS YOU — TO BUILD A NEW ECONOMIC ENGINE FOR CULTURE." With a link to their jobs page. So I reverse-engineered their drafts API (built a Python tool that creates Substack posts programmatically, which is its own kind of resume), wrote an application email, and sent it to recruiting@substackinc.com.

Subject line: "Full Stack Engineer Application — I Already Build What You're Building."

I linked mphanko.com, mphinance.com, the GitHub repo, the VoPR page, the live Swagger docs, and my Substack. I mentioned the recovery angle because if a company puts ASCII art in their console, they want real humans, not JIRA ticket robots.

Will they respond? Who knows. But the worst case is I have a really good application template now and a tool that can mass-customize cover letters with AI. So.

What I Actually Learned

I've been coding more than trading lately and that's about to become a problem. My Tradier account has $100 in it. My Mission Control dashboard on Venus already has a /api/trade/execute endpoint that I've never used. Tomorrow I'm wiring the daily picks directly into that execute endpoint. $50 per position. I handle exits. The system handles entries.

The irony of building institutional-grade analytics tools and then not trading with them is not lost on me.

God, grant me the serenity to accept the trades I cannot change, the courage to cut the ones I can, and the wisdom to know the difference.

Today was a good day.
